## Theory

Finite automata are mathematical models used to recognize regular languages. A finite automaton consists of a finite set of states, an input alphabet, a transition function, a start state, and a set of accepting states. Depending on the transition function, automata can be deterministic (DFA) or nondeterministic (NDFA).

Nondeterministic finite automata allow multiple possible transitions for the same state and input symbol. Despite this flexibility, every NDFA has an equivalent DFA that recognizes the same language. Finite automata are equivalent in expressive power to regular grammars, meaning that any language generated by a regular grammar can be recognized by a finite automaton.

## Objectives

The objective of this laboratory work is to implement a finite automaton programmatically using Variant 26, determine whether the automaton is deterministic, convert the NDFA into an equivalent DFA, transform the automaton into a regular grammar, classify the grammar according to the Chomsky hierarchy, and verify whether given strings belong to the language recognized by the automaton.

---

## Implementation Description

The program defines a class that represents a finite automaton by storing its set of states, alphabet, transition function, start state, and set of final states. The automaton described in Variant 26 is nondeterministic because some transitions lead to multiple possible next states.

A method checks whether the automaton is deterministic by verifying that each state and symbol pair leads to exactly one next state. Another method converts the automaton into an equivalent regular grammar by interpreting states as nonterminals and transitions as production rules. Final states produce epsilon, allowing termination of derivations.

To eliminate nondeterminism, the subset construction algorithm is used to convert the NDFA into a DFA. In this process, each DFA state represents a set of NDFA states. The resulting DFA is guaranteed to be deterministic and to recognize the same language.

An additional method simulates the automaton to determine whether input strings are accepted. The program processes each symbol of a word and updates the set of current states according to the transition function. A word is accepted if at least one final state is reached after processing all symbols.

The main program builds the automaton, displays its components, performs the NDFA to DFA conversion, classifies the derived grammar, and tests several example strings.

## Results

The automaton defined in Variant 26 is nondeterministic because the transition from state q0 on symbol a leads to two possible states. After applying the subset construction method, an equivalent deterministic automaton is obtained.

The conversion to a regular grammar produces a Type-3 grammar according to the Chomsky hierarchy, confirming that the recognized language is regular.

Tested words demonstrate correct behavior of the automaton: words that follow the transition rules and reach the final state are accepted, while others are rejected.

## Conclusions

The implementation successfully models the finite automaton described in Variant 26 and demonstrates key theoretical concepts from automata theory. The program verifies nondeterminism, constructs an equivalent DFA, derives a corresponding regular grammar, and tests language membership for input strings.

The results confirm the theoretical equivalence between nondeterministic and deterministic finite automata and between finite automata and regular grammars. Therefore, the laboratory work illustrates how regular languages can be represented and analyzed using computational models.
